# Architectural Decision Records

This directory tracks high-impact technical and operational decisions.

## When an ADR is required

- Changes to auth/security behavior
- Portfolio/risk policy changes
- Execution/broker integration changes
- Backtest assumptions used for promotion decisions
- Governance workflow changes (promotion gate, artifact policy, CI gates)

## Naming convention

Use `ADR-XXXX-short-title.md` (example: `ADR-0001-portfolio-turnover-cap.md`).

## Status lifecycle

- `Proposed`: under discussion
- `Accepted`: approved and in effect
- `Superseded`: replaced by a newer ADR
